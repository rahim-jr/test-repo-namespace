=== BACKEND API TEST DATA (AUTH + PRODUCT CRUD) ===

Base URL: http://localhost:5000/api

Notes:
- The backend exposes authentication endpoints under `/api/auth`.
- Creating, updating, and deleting products require a valid JWT in the `Authorization` header: `Authorization: Bearer <TOKEN>`.
- Replace `REPLACE_WITH_ACTUAL_ID` with the real product _id returned by the create product response.

---

AUTHENTICATION
--------------

1. REGISTER
   POST http://localhost:5000/api/auth/register

   Request body (JSON):
   {
     "name": "Test User",
     "email": "test@example.com",
     "password": "TestPass123!"
   }

   Expected response:
   {
     "success": true,
     "token": "<JWT_TOKEN>",
     "user": { "_id": "...", "name": "Test User", "email": "test@example.com" }
   }

2. LOGIN
   POST http://localhost:5000/api/auth/login

   Request body (JSON):
   {
     "email": "test@example.com",
     "password": "TestPass123!"
   }

   Expected response:
   {
     "success": true,
     "token": "<JWT_TOKEN>",
     "user": { "_id": "...", "name": "Test User", "email": "test@example.com" }
   }

3. GET CURRENT USER (protected)
   GET http://localhost:5000/api/auth/me

   Headers:
     Authorization: Bearer <JWT_TOKEN>

   Request body: (none)

   Expected response:
   {
     "success": true,
     "user": { "_id": "...", "name": "Test User", "email": "test@example.com", "createdAt": "..." }
   }

QUICK TEST SEQUENCE (brief)
---------------------------
1) Register or login to get a token.
2) Use the token to create products.
3) Retrieve the list with GET /.
4) Use GET /:id, PUT /:id, DELETE /:id as needed.

REPLACEMENTS
------------
- Replace `<JWT_TOKEN>` with the actual token returned on login/register.
- Replace `REPLACE_WITH_ACTUAL_ID` with the product `_id` returned from create responses.

NOTES
-----
- If you change the product model to include `user` (owner), created products will be associated with the authenticated user and update/delete will only succeed for the owner.
- For automated tests / CI, you can generate a user and record the token programmatically, then run the product CRUD calls with that token.
